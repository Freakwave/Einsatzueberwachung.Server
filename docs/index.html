Architektur

Durchdachtes Design

Klare Schichtentrennung

Die Anwendung folgt einer klaren Service-orientierten Architektur mit getrennten Verantwortlichkeiten für Persistenz, Echtzeit-Kommunikation und UI.

  • Blazor Razor Components als reaktive UI-Schicht
  • REST-API + SignalR Hub für alle Clients
  • Hosted Services für Background-Tasks
  • Domain-Events → SignalR Relay Pattern
  • Separate RuntimeDbContext für flüchtigen State
  • JSON-Dateien für Stammdaten & Konfiguration
